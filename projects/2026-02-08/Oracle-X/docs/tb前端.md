# Oracle-X 前端MVP：最大化展示核心价值的设计方案

你的核心不是"做了一个好看的交易界面"——市面上有无数交易界面比你好看。你的核心是**"在用户做出交易决策的那一刻，用AI拦截并提供多维分析"**。前端的每一个像素都应该服务于这个叙事。

---

## 一、砍掉什么、保留什么

先说结论：**前端只需要一个页面，三个状态。**

### 砍掉的（不做）

| 功能             | 砍掉理由                            |
| ---------------- | ----------------------------------- |
| 用户登录/注册    | 黑客松Demo不需要账户体系            |
| 钱包连接（真实） | MVP不上链，模拟即可                 |
| 交易对搜索       | 硬编码3个交易对下拉选择             |
| K线图/深度图     | 你不是在做TradingView，不要抢它的戏 |
| 历史交易记录     | 与核心无关                          |
| 设置页面         | 不需要                              |
| 响应式/移动端    | 只做桌面端演示                      |

### 保留的（必须做到极致）

| 功能                      | 它展示的核心价值           |
| ------------------------- | -------------------------- |
| 实时价格 + 关键指标       | "我们接入了真实市场数据"   |
| 做多/做空按钮             | "交易决策的入口"           |
| **拦截弹窗 + 流式AI分析** | **这是你的全部**           |
| 用户画像卡片（硬编码）    | "我们理解每个交易者的个性" |
| 最终建议Badge             | "明确的、可执行的结论"     |

---

## 二、三个状态的设计

整个前端只有三个状态，评委在30秒内就能理解你在做什么：

### 状态一：待决策（Idle）

这是用户打开页面看到的默认状态。它的作用不是"展示行情"，而是**营造交易场景**——让评委立刻理解"这个人正准备交易"。

```
┌──────────────────────────────────────────────────┐
│  🔮 Oracle-X                    ETH/USDT ▾       │
├──────────────────────────────────────────────────┤
│                                                  │
│         $ 3,842.50                               │
│         ▼ -2.35%  (24h)                          │
│                                                  │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐   │
│  │ RSI (14)   │ │ MACD       │ │ 市场情绪    │   │
│  │   72.3     │ │  ▲ 金叉    │ │  25/100    │   │
│  │  ● 超买    │ │  Bullish   │ │ 极度恐惧   │   │
│  └────────────┘ └────────────┘ └────────────┘   │
│                                                  │
│  ┌──────────────────────────────────────────┐    │
│  │ 👤 Your Profile                          │    │
│  │ Swing Trader · Win Rate: L62% S41%       │    │
│  │ Risk: Medium · 147 txns analyzed         │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│     [ 🟢 LONG ]            [ 🔴 SHORT ]         │
│                                                  │
└──────────────────────────────────────────────────┘
```

**关键设计细节：**

**指标卡片不是装饰，是"弹药"。** 这三张卡片展示的是 `trading-signals` 库实时计算出来的真实指标。它们的存在是为了在Demo时让你说出这句话："这些指标是实时计算的，不是AI编的。AI拿到的是真实数据，基于真实数据做分析。" 每张卡片不仅显示数值，还显示一个**信号状态标签**（超买/金叉/极度恐惧），这个标签用颜色编码（红/绿/黄），让评委一眼就能感知市场状态。

**用户画像卡片是你的"秘密武器"。** 虽然是硬编码的数据，但它在视觉上传递了一个强烈的信号：这个产品认识我。它知道我的交易风格、我的胜率、我的风险偏好。当AI分析中引用这些数据时（"根据您的历史数据，做空胜率仅41%"），评委会产生"哇"的感觉。这张卡片用一个微妙的渐变边框或图标来暗示"这是从链上数据分析出来的"，不需要说破，让人自己去理解。

**按钮是全页面最大的视觉元素。** 它们不是普通按钮，它们是"触发器"。用足够大的尺寸、足够强的颜色对比，让评委的注意力自然被引导到这里。按钮上可以加一行小字："Oracle-X will analyze before execution"——这一行字就把你的产品定位讲清楚了。

---

### 状态二：分析中（Analyzing）— 这是你的高光时刻

用户点击 LONG 或 SHORT 后，整个页面的焦点转移到分析弹窗。**这个弹窗的设计质量直接决定了你的Demo成败。**

```
┌──────────────────────────────────────────────────┐
│  ◉ Oracle-X Analysis                          ✕  │
│  ETH/USDT · LONG · $3,842.50                    │
│  ─────────────────────────────────────────────── │
│                                                  │
│  ⏱ Analyzing 3 dimensions...                    │
│  [████████████░░░░░░░░] Technical ✓              │
│  [████████░░░░░░░░░░░░] Market Sentiment ●       │
│  [░░░░░░░░░░░░░░░░░░░] Personal Risk ○          │
│                                                  │
│  ─────────────────────────────────────────────── │
│                                                  │
│  【趋势分析】                                     │
│  当前ETH处于4小时级别的下降通道中，RSI(14)         │
│  读数72.3已进入超买区域。MACD虽然刚形成金叉，      │
│  但histogram柱状图力度较弱，上攻动能不足...▊       │
│                                                  │
│  【波动性评估】                                   │
│  布林带宽度正在收窄，当前价格紧贴上轨（95%         │
│  位置），这通常预示着...▊                         │
│                                                  │
│  (流式输出中，文字逐步出现)                       │
│                                                  │
└──────────────────────────────────────────────────┘
```

**关键设计细节：**

**顶部的"三维度进度条"是纯粹的叙事工具。** 它不需要反映真实的计算进度（AI是一个统一的Prompt，不是分三步跑的）。但它在视觉上传递了一个关键信息：Oracle-X不是简单地问了一下ChatGPT，它在做**多维度、结构化的分析**。技术分析 → 市场情绪 → 个人风险，这三个维度的名字本身就是你的产品差异化。实现方式很简单：流式文本中检测到"趋势分析"关键词时，第一个进度条跑满；检测到"市场情绪"时，第二个跑满；以此类推。

**流式文本是"信任建造器"。** 用户看着文字一个字一个字地出现，会产生"AI正在思考"的感觉，这比瞬间弹出一大段文字更有说服力。文字末尾的闪烁光标（`▊`）强化了这种"正在进行中"的感知。每个分析维度用【】标题分隔，让结构一目了然。

**引用真实指标数据。** 分析文本中出现的"RSI(14) 72.3"、"布林带上轨95%位置"这些数字，和上一个页面中指标卡片显示的数字是一致的。评委如果注意到这一点，会立刻意识到：AI不是在胡说，它在引用真实计算结果。这种前后一致性是建立信任的关键。

---

### 状态三：决策（Verdict）— 分析完成后的最终画面

流式输出结束后，弹窗底部"生长"出最终结论区域：

```
│  ...建议谨慎考虑该方向的交易。                    │
│                                                  │
│  【个人风险提示】                                 │
│  根据您的历史交易数据，您在做空方向的胜率（41%）   │
│  显著低于做多方向（62%）。结合当前市场的高波动     │
│  环境，该交易的预期风险收益比不理想。              │
│                                                  │
│  ═══════════════════════════════════════════════  │
│                                                  │
│  ┌──────────────────────────────────────────┐    │
│  │         🔴 HIGH RISK WARNING             │    │
│  │                                          │    │
│  │   Market conditions and your personal    │    │
│  │   trading profile suggest caution.       │    │
│  │                                          │    │
│  │   Confidence: 78%                        │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  [ ✅ Execute Anyway ]     [ ← Back to Panel ]   │
│                                                  │
└──────────────────────────────────────────────────┘
```

**关键设计细节：**

**结论Badge必须是全页面最醒目的元素。** 三种颜色（🟢绿/🟡黄/🔴红）对应三种建议，用大色块背景而不是小标签。评委在5米外都应该能看清当前是什么建议。这个Badge的出现应该有一个短暂的动画（从无到有的淡入+轻微放大），制造一个"揭晓答案"的仪式感。

**"Confidence: 78%"这个数字是点睛之笔。** 它暗示系统不是简单地给出"好/坏"的二元判断，而是有一个量化的置信度。这个数字可以从Prompt中让AI输出，也可以基于指标信号的一致性程度简单计算（比如3个指标中2个看空 → 67%置信度看空）。

**"Execute Anyway"按钮的措辞是故意的。** 它传递了Oracle-X的核心哲学：我给你建议，但最终决策权在你手上。这不是一个自动交易机器人，这是一个Copilot。这个按钮在Demo中你不需要真的点它（点了之后显示一个"Order Submitted"的Toast就行），但它的存在完成了整个用户故事。

## 三、Demo时的"Wow Moment"编排

前端做到上面的程度后，你的Demo应该这样走：

**第一幕（10秒）：** 打开页面，评委看到实时跳动的ETH价格和指标卡片。你说："Oracle-X接入了实时市场数据，并用技术分析库计算了关键指标。"

**第二幕（5秒）：** 指向用户画像卡片。你说："系统已经分析了这位交易者的链上历史，他是一个中频Swing Trader，做多胜率62%，做空只有41%。"

**第三幕（5秒）：** 点击 SHORT 按钮。你说："现在他想做空ETH。Oracle-X会在交易执行前拦截这个请求。"

**第四幕（15-20秒）：** 分析弹窗出现，文字流式输出。你**不说话**，让评委自己读。沉默是最好的演示——它意味着你的产品自己会说话。

**第五幕（5秒）：** 红色警告Badge出现。你说："Oracle-X综合了技术面、市场情绪和这位交易者的个人数据，给出了高风险警告。注意这里——它特别提醒了这位交易者做空的历史胜率只有41%。这种个性化的风险提示，是现有任何工具都做不到的。"

**总时长：40-45秒。** 干净利落。

## 四、视觉优先级排序

如果时间不够，按以下优先级砍UI打磨的工作：

| 优先级 | 元素                           | 时间预算 | 砍了的后果             |
| ------ | ------------------------------ | -------- | ---------------------- |
| P0     | 分析弹窗（流式文本+结论Badge） | 60min    | 没有Demo，项目死亡     |
| P1     | 做多/做空按钮 + 点击触发逻辑   | 20min    | 无法触发核心流程       |
| P1     | 实时价格显示                   | 20min    | 失去"真实数据"的说服力 |
| P2     | 指标卡片（RSI/MACD/FGI）       | 30min    | 分析看起来像纯AI幻觉   |
| P2     | 用户画像卡片                   | 15min    | 失去最大差异化卖点     |
| P3     | 深色主题 + 品牌感              | 20min    | 看起来像作业而不是产品 |
| P3     | 三维度进度条动画               | 15min    | 少了一层叙事，但不致命 |
| P4     | 交易对切换                     | 10min    | Demo时只用ETH也行      |
| P4     | 价格闪烁动画                   | 5min     | 纯锦上添花             |

**如果只剩3小时给前端：** 做P0 + P1 + P2，跳过P3和P4。一个功能完整但丑的Demo，远好过一个漂亮但跑不通的半成品。

---

## 五、一句话总结

你的前端不是在做一个交易所，你在做一个**30秒的故事**：一个交易者准备做空 → AI拦截 → 多维分析 → 发现这个人不擅长做空 → 红色警告。评委记住的不是你的UI多好看，而是这个故事有多清晰、多有说服力。前端的每一个元素要么在推进这个故事，要么就不应该存在。
