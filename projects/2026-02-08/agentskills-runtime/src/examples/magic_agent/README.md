# Magic Agent (通用助手模板)

## 📖 项目名称

`magic_agent`

## 🚀 功能简介

这是一个用于创建强大通用助手的模板示例。它展示了一种更复杂的 Agent 架构，包括一个精心设计的、用于指导 Agent 行为的系统提示词，以及一个“思考”工具，让 Agent 的推理过程更加明确和可控。

## ✨ 核心特性

- **`tool-loop` 执行器**: 使用一种没有明确 `Thought` 标签但仍然支持多步工具调用的循环执行器。
- **工具过滤**: 通过设置 `enableToolFilter: true`，让 Agent 在执行前根据问题智能筛选可能用到的工具，提高效率。
- **元工具 (Meta-tool)**: 定义了一个 `thinking` 工具，它不执行具体任务，而是让 Agent 把自己的思考过程作为一步操作记录下来，有助于调试和规划复杂任务。
- **高级提示词工程**: 提供了一个非常详细和结构化的系统提示词，定义了 Agent 的工作循环和行为准则。

## 🔧 如何运行

1.  **配置 API Keys/MCP 服务**: 根据你在 `@agent` 的 `tools` 属性中配置的 MCP 服务，确保对应的 API Key 或服务 URL 已正确设置。
2.  **运行项目**: 在终端中执行以下命令，并向它提出一个需要多步操作才能解决的问题。
    ```bash
    cjpm run --name magic.examples.magic_agent
    ```
